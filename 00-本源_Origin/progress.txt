# 道_抱一_带传动 · 进展笔记

## 2026-04-22  racetrack extrude 闭合修复 (核心 bug)

### 根因
SW API `SketchManager.CreateLine` / `CreateArc` 每次创建**新的**端点 SketchPoint,
即使坐标完全重合, SW 仍视为两个独立顶点; `ISketch.GetSketchContours[0].IsClosed`
返回 `False`, `FeatureExtrusion3` 返 `None`.

对比: `CreateLine` 4 次画矩形可 extrude, 因 SW 对**连续 line-line 端点**有
auto-inferencing 自动生成 Coincident. 但对 **line-arc-line-arc** (racetrack)
或 2 opposing arcs 对侧拓扑, inferencing 失效 → 留下 1 个未合并缺口.

### 修复 (道_抱一_带传动.py · _sketch_racetrack)
绘制 4 段后, 对每对相邻端点显式合并:

```python
model.ClearSelection2(True)
seg_a.GetEndPoint2.Select4(False, NULL_SELDATA)
seg_b.GetStartPoint2.Select4(True, NULL_SELDATA)
if sm.GetSelectedObjectCount2(-1) >= 2:
    model.SketchAddConstraints("sgMERGEPOINTS")
```

关键点:
- `Select4` 需要 `SelectData` 而非 None → 用 `VARIANT(VT_DISPATCH, None)` 封装
- 若 SW 已自 inference 合并, 两次 Select 命中同一点 → count=1 → 跳过 (no-op)
- 只需 1 对手动 merge (通常是最后 wraparound), 其余 3 对自动

### 验证
- `selftest` 全通过 (向量/主轴/平面/UV/racetrack/signature/plan)
- `compute_belt_racetrack` path 4 seg, gap=0.0 (坐标层面), path[3]→[0] 需 sgMERGEPOINTS
- `forge_part` 14.2s 成功锻造 outer + inner cut (racetrack + racetrack-cut)
- `install` 5 strands 入装配, save 耗 5+ 分钟 (大装配 70+ components 正常)

对齐到 `(x, y, z)`, **非 part origin**. 对 origin 不在 bbox 中心的 part 造成轴向偏移.
belt_a45d9854e2 part bbox Z=[-45,+720] (原点在 drive 侧), 几何中心 Z=+337.5.

### API 追加备忘
- `face.Select4(append, SelectData)` **只对 `comp.GetBody` 拿的 face 生效**.
  `GetModelDoc2.GetBodies2` 返的是 part 级 face, 装配上下文不可选.
- `AddMate5(err_status=1)` **不代表失败**. mate 非 None 即创建成功, err=1 常为警告
  (如已冗余/位置微偏). 验证需 `probe_mates` + `error_status in (None, 0)`.
- `ISurface.CylinderParams` 返 7 tuple: `(root.x, root.y, root.z, axis.x, axis.y, axis.z, R)` 米.
  root 是**轴线上任一点**, 非端点; axis 是**单位向量**, 方向可为正可为负 (需抽象 ±)
- `ISurface.Identity` ∈ {4001=PLANE, 4002=CYLINDER, 4003=CONE, 4004=SPHERE, ...}
- Concentric mate 允许轴向滑动 + 绕轴旋转. 欲完全定义需再加:
  · 端面 Distance/Coincident (锁轴向)
  · 侧面 Parallel/Angle (锁旋转)
