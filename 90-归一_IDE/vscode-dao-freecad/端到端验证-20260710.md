# 归一插件 · 四板块端到端验证 (2026-07-10)

> 环境: Devin VM (Ubuntu 22.04) · FreeCAD 0.19 · xpra :100 (HTML5 :14500) · 桥接 :18920 · Devin Desktop 1.110.1 扩展宿主 · Node v20

四板块（FreeCAD 本源 / 归一工作台 / Devin Desktop 宿主 / Proxy Pro）在 Devin Desktop 扩展宿主内全链路联通，`/shell` 归一外壳四个 board 全部可用，看门狗自愈实测通过。

### 3. 生命周期与自愈

- 激活: `onStartupFinished` 自动起 shell(9920)/探活桥接, 端口占用自动后退(tryListen +1, 最多 9 次)。
- 看门狗: kill 桥接进程后, 15s 周期探活 → 自动重拉 FreeCAD, 实测约 40s 内 `bridge.ok=true` 恢复, 指数退避 4/8/16 生效。

### 4. Proxy Pro / dao-ai-base

- Proxy Pro 反代端口 8985 在宿主内监听; 未启用反代时 `/v1/models` 返回明确 503 `revproxy_disabled`（优雅降级, 不影响其余板块）。
- 渠道配置/模型路由面板正常渲染, 通道列表与探测按钮齐备。
- dao-ai-base: `node --check` 通过; 宿主外加载报 `Cannot find module 'vscode'` 属预期(仅限扩展宿主内加载)。

## 已知边界

- Devin Desktop 首启有账号登录门(OAuth), 不影响扩展宿主激活与 /shell 全部功能。
- VM 无 GPU: 工作台视口 WebGL 不可用时按设计降级(模型树/属性/控制台可用), FreeCAD 整窗经 xpra 软渲染正常。
