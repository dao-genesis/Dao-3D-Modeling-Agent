## 一、当前架构判定 (是什么)

### 1. 五层闭环 + 万法门面 (成立且实测贯通)


### 2. 交互模式 (怎么用)

- **Agent 通路**: `cad_agent.mcp_server` stdio JSON-RPC, tools/list 240 工具; Devin/任意 MCP 客户端可直接驱动真 FreeCAD 内核。本会话即以 Devin 为核心智体带入实测。
- **人类通路**: `from 万法 import 道` 单一门面 + CLI + FreeCAD GUI Workbench 对话面板 + VS Code 归一工作台。
- **反向优先**: 意→先搜天下已有(20平台)→适配; 无有→本源构建。

### 3. 本会话新增核验事实

- FEM own-deck 路径 (PR #10) 在本 VM 复现有效: 全量测试含 FEM 悬臂梁 0 失败。

## 二、对照业界优质 3D 建模实践的解构 (差在哪)

| 维度 | 业界标杆 | 本项目现状 | 判定 |
|---|---|---|---|
| 几何内核 | OCCT/Parasolid B-Rep | OCP/OCCT 直连 + FreeCAD 双内核桥 | ✅ 同源 |
| 参数化历史 | 特征树 + 依赖图重算 | ops 线性序列 + _link 依赖替换 | ⚠️ 无真依赖图, 改参重放为整链重跑 |
| 草图约束求解 | 独立 2D 求解器 (solvespace/PlaneGCS) | 重放借 FreeCAD PlaneGCS; 自身无求解器 | ⚠️ 脱离 FreeCAD 即失约束语义 |
| 装配 | mate 图 + 全局 DOF 求解 + 运动学联动 | asm.dof 逐件核算 (PR #8), 无全局闭环求解 | ⚠️ 半成品 |
| 验证 | 干涉/质量/公差/FEM 闭环 | 八层核审 + FEM own-deck + 干涉 | ✅ 特色强项 |
| 版本化 | 模型级 diff/merge (类 git) | 仅文件级 git | ❌ 缺 |
| 代码化 CAD | CadQuery/build123d 生态 | code.run 已接 CadQuery (PR #7); build123d 与 OCP 版本錯位 | ⚠️ 半通 |

## 三、内生工程问题 (自身的病)

1. **巨文件**: freecad_ops.py 5053 行/139 函数、docformat.py 4389 行、dao_solidworks.py 7039 行 — 单文件即单点, 无法并行演进。
2. **多副本漂移**: freecad_backend.py 存在 4 份 (10-反笙/cad_agent/80-智体/90-归一×2), 仅 10-反笙 版含 sketch 重放; fc_reverse 单份但 IDE 内嵌 dao 运行时靠手工 cp 同步。应改为单源 + 构建期复制。
3. **automerge 过激**: 内核级变更 <2 分钟自动合并, 无人审窗口。
4. **CI 环境 ≠ 实机环境**: CI 走 cadquery-ocp 无 FreeCAD, FreeCAD 相关路径仅本地可测; 0.19↔1.x 兼容层 (PR #9) 缺 1.x 实机回归。
