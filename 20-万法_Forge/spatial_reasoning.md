# 空间推理层 — Spatial Reasoning Substrate

> **反者道之动** — 人先理解几何，再动手建模。Agent亦然。
> 此文件是Agent的"空间直觉"——在写任何代码之前，先在此层完成几何推理。

---

## 一、根因：为何Agent不能像人一样理解3D

人类3D认知建立在三根柱子上（Shepard/Gibson/Marr）：

| 人类认知 | 机制 | Agent当前缺失 |
|----------|------|---------------|
| **心理旋转** (Shepard & Metzler 1971) | 内部表征可连续几何变换 | Agent无内部3D表征，只有文字→代码→结果 |
| **可供性感知** (Gibson 1979) | 看到物体即知用途：孔→插入，面→支撑，臂→弹性 | Agent不从功能反推结构 |
| **2.5D草图** (Marr 1982) | 从视角提取深度、轮廓、遮挡关系 | Agent不能从图片提取3D拓扑 |

**根本缺陷**：当前Agent是"命令调度器"——收到文字→生成代码→运行→检查输出。
它在写代码前**不做几何推理**。人类工程师在画第一条线前，已经在脑中完成了：
- 这个圆角半径不能超过最短边的一半
- 这个壁厚在悬臂处需要加厚
- 这个孔的位置必须避开应力集中区
- 这个零件需要分成壳体+盖板才能FDM打印

## 二、几何约束直觉（写代码前的预检）

### 2.1 圆角/倒角约束

```
fillet_max = min(adjacent_edge_lengths) / 2 - tolerance
chamfer_max = min(adjacent_edge_lengths) / 2 - tolerance

规则：
- fillet(r) 必须 r < 所有相邻边长/2
- 两个相邻fillet的半径之和 ≤ 它们共享边的长度
- 在compound/boolean结果上做fillet极易失败 → 先fillet再boolean
- build123d的fillet比CadQuery更脆弱 → 复杂体优先用CadQuery
```

### 2.2 Boolean运算约束

```
规则：
- 减去的体积必须与被减体有交集（否则无效果）
- 工具体应略大于目标体（穿透1mm确保完全切割）
- FreeCAD的cut在Compound上不可靠 → 先fuse所有工具体再单次cut
- 布尔后检查: 结果面数应增加（切割）或减少（融合）
- 非流形输入导致布尔失败 → 先quality检查
```

### 2.3 壁厚约束

```
最小壁厚（与工艺绑定）：
- FDM: ≥ 1.2mm（单壁0.4喷嘴×3道）, 推荐 ≥ 1.5mm
- SLA: ≥ 0.5mm, 推荐 ≥ 0.8mm
- CNC: ≥ 1.0mm（刀具限制）
- 注塑: ≥ 0.8mm（流动性）, 均匀（避免缩水）

壁厚计算：
- shell厚度 = (外径 - 内径) / 2
- 悬臂处壁厚 × 1.5（补偿层间结合弱）
- 螺丝孔周围壁厚 ≥ 螺丝直径（M3孔需≥3mm壁）
```

### 2.4 孔与配合约束

```
标准孔径（M系列螺丝通孔）：
- M2: 2.4mm  M2.5: 2.9mm  M3: 3.4mm  M4: 4.5mm  M5: 5.5mm  M6: 6.6mm
- 自攻孔(FDM): 螺丝直径 - 0.3mm
- 紧配合(FDM): 标称 + 0.1mm
- 间隙配合(FDM): 标称 + 0.2~0.3mm
- 过渡配合: 标称 + 0.15mm

孔深约束：
- 通孔: 深度 = 壁厚 + 2mm（穿透余量）
- 盲孔: 深度 ≤ 壁厚 - 最小壁厚
- 沉头孔: 沉头深度 = 螺丝头高度 + 0.2mm
```

### 2.5 悬臂与支撑约束

```
FDM无支撑规则：
- 悬臂角度 ≤ 45°（相对垂直面）
- 桥接距离 ≤ 10mm（无支撑跨越）
- 悬臂超过45° → 设计自支撑结构或拆件
- 底面需平坦 ≥ 10mm²（第一层附着）

打印方向推理：
- 最大平面朝下 → 最少支撑
- 应力方向 ⊥ 层线方向 → 最强
- 孔轴线垂直于打印方向 → 最圆
```

## 三、功能→结构映射（可供性推理）

人类看到"杯子"就知道：空心容器+手柄+平底。
Agent需要同样的映射能力：

### 3.1 功能原型库

| 功能需求 | 结构模式 | 关键参数 |
|----------|----------|----------|
| **容纳** (contain) | 壳体 = 外壁 - 内腔 | 壁厚, 内径, 深度 |
| **支撑** (support) | 底板 + 立柱/肋 | 底面积, 高度, 肋间距 |
| **连接** (fasten) | 孔 + 凸台/卡扣 | 孔径, 壁厚, 卡扣弹性臂长 |
| **导向** (guide) | 槽 + 导轨 | 槽宽=轨宽+间隙, 深度 |
| **密封** (seal) | O型圈槽 + 唇边 | 槽宽=线径×1.4, 槽深=线径×0.7 |
| **散热** (ventilate) | 栅格/百叶窗 | 开口率≥30%, 最小孔宽≥1.5mm(FDM) |
| **铰接** (hinge) | 活页轴 + 圆弧面 | 轴径, 间隙0.3mm |
| **弹性** (flex) | 薄臂/蛇形弹簧 | 臂长≥10×壁厚, 壁厚≤1mm |
| **堆叠** (stack) | 凸台 + 凹槽定位 | 凸台高2mm, 间隙0.15mm |
| **展示** (display) | 倾斜面 + 槽 | 角度60-75°, 唇高≥5mm |

### 3.2 拆件决策树

```
零件能一体打印?
  ↓
最大悬臂 > 45°且无法旋转打印方向? ──YES──→ 拆为上下件(卡扣/螺丝连接)
  ↓ NO
存在内部腔体且无开口? ──YES──→ 拆为壳体+盖板
  ↓ NO
存在不同材料/颜色需求? ──YES──→ 拆为多件
  ↓ NO
最大尺寸超过打印机热床? ──YES──→ 拆为拼接件(榫卯/螺丝)
  ↓ NO
一体打印 ✓
```

## 四、拓扑推理（形状的本质理解）

### 4.1 Euler公式验证

```
V - E + F = 2  (对于单连通封闭多面体)
V - E + F = 2 - 2g  (g = 亏格/孔数)

应用：
- 一个孔(如杯柄): g=1, V-E+F=0
- 两个孔: g=2, V-E+F=-2
- 简单实体: g=0, V-E+F=2

验证方法: forge_v3.py measure → euler_number
如果euler_number ≠ 预期 → 几何有缺陷
```

### 4.2 对称性利用

```
识别对称性 → 减少工作量 + 保证一致性：

- 双边对称(bilateral): mirror() / .mirror()
- 旋转对称(radial): rotate_extrude() / revolve()
- 平移对称(translational): linear_extrude() / for循环
- 螺旋对称: screw() / helix sweep

原则: 永远利用对称性建模，不手动复制几何
```

### 4.3 尺寸链推理

```
从外到内推导:
总长 = 壁厚_左 + 内腔 + 壁厚_右
→ 内腔 = 总长 - 2×壁厚

从功能到几何:
要放入的物体尺寸 + 间隙 = 内腔尺寸
内腔尺寸 + 2×壁厚 = 外壳尺寸

从配合推导:
轴径 + 2×间隙 = 孔径
孔径 + 2×最小壁厚 = 凸台外径
```

## 五、失败预测（在执行前知道什么会出错）

### 5.1 常见失败模式

| 操作 | 失败条件 | 预防 |
|------|----------|------|
| `fillet(r)` | r ≥ 最短相邻边/2 | 计算最大可用半径 |
| `chamfer(d)` | d ≥ 最短相邻边/2 | 同上 |
| `shell(-t)` | t ≥ 最小尺寸/2 | 检查最薄处 |
| `hole(d)` | d ≥ 面的最短边 | 检查面尺寸 |
| `cutThruAll()` | 截面与体无交集 | 验证截面位置 |
| `fuse()` then `cut()` | Compound上cut失败 | fuse工具体后单次cut |
| `sweep(path)` | 路径自交/截面过大 | 截面半径 < 路径最小曲率半径 |
| `loft(sections)` | 截面顶点数不同 | 统一截面顶点数 |
| `revolve()` | 截面跨越旋转轴 | 截面完全在轴的一侧 |
| `export_stl` | 体积为0/非流形 | 先quality检查 |

### 5.2 引擎特定陷阱

```
CadQuery陷阱:
- slot2D().cutThruAll() → 某些情况BREP内核失败 → 用rect()替代
- 链式.faces(">Z")在复杂体上选错面 → 用.faces(">Z")[-1]明确
- FutureWarning: save将被移除 → 用exporters.export()

build123d陷阱:
- fillet/chamfer在compound体上极易失败 → 在union前做
- export_stl是函数不是方法 → export_stl(part, path)
- sweep需要显式path参数 → sweep(path=wire)
- Locations用元组不是列表

OpenSCAD陷阱:
- sphere()+hull() 极慢 → cube()+hull()
- minkowski() 极慢 → 手动倒角
- $fn>128 指数慢 → $fn=64足够
- 布尔嵌套>3层 → 拆分

FreeCAD陷阱:
- Compound上的cut/fillet不可靠 → fuse后removeSplitter()
- freecadcmd路径有空格需引号
- makeFillet对某些边失败(尖角) → try/except降级到chamfer
```

## 六、空间推理流程（融入步骤2规划）

**在写任何建模代码之前，必须完成以下推理：**

### 6.1 几何可行性预检 (Geometric Preflight)

```
输入: 用户需求 + 初步参数
输出: 可行/不可行 + 修正建议

检查清单:
□ 所有fillet半径 < 相邻最短边/2?
□ 壁厚 ≥ 工艺最小值?
□ 孔径 ≤ 面尺寸?
□ Boolean工具体与目标体有交集?
□ 尺寸链闭合(外部 = 内部 + 2×壁厚)?
□ 悬臂角度 ≤ 45°(FDM)?
□ 打印方向可行?
```

### 6.2 结构分解 (Part Decomposition)

```
输入: 功能需求
输出: 子部件清单 + 连接方式

方法:
1. 识别功能原型(容纳/支撑/连接/...)
2. 映射到结构模式
3. 确定拆件策略
4. 定义接口(配合面/孔位/卡扣)
```

### 6.3 约束传播 (Constraint Propagation)

```
当一个参数改变时，推导所有受影响参数:

例: 壁厚 3mm → 2mm
→ 内腔增大 2mm
→ 凸台外径减小 2mm → 检查凸台强度
→ fillet最大半径可能变化
→ 重量减少 → 重心偏移
→ 刚度降低 → 悬臂处可能需要加肋
```

## 七、为道日损 — 简化原则

```
1. 能一个操作完成的，不用两个
2. 能用对称性的，不复制几何
3. 能参数化的，不硬编码
4. 能在设计时约束的，不在验证时修复
5. 能用简单几何近似的，不用复杂曲面
6. 先让它工作，再让它精确，最后让它美观
```

**最小可行几何** (Minimum Viable Geometry):
- 第一轮: 基本形状 + 关键特征 (box + holes)
- 第二轮: 倒角/圆角 + 配合细节
- 第三轮: 美观 + 制造优化
- 不要一步到位，分层递进

## 八、验证：Agent是否真的在"想"

在每次建模前，Agent应能回答：
1. 这个物体的**拓扑**是什么？(亏格、面数预估)
2. 最可能**失败的操作**是哪个？
3. 最**关键的尺寸约束**是什么？
4. 如果用手拿起这个物体，**感觉**如何？(重量、手感、重心)
5. 这个物体**不能**做什么？(负空间思维)

如果Agent不能回答这五个问题，它还没有真正理解这个三维物体。

## 九、八层审核系统 — 认知→代码的映射

`dao_audit.py` 将上述空间推理形式化为可执行的八层审核:

| Layer | 名称 | 认知来源 | 核心算法 |
|-------|------|---------|---------|
| 0 | 拓扑完整性 | Euler直觉 (§六) | OCCT BRepCheck |
| 1 | 几何健全性 | 心理旋转 (Shepard) | 体积/面积/质心一致性 |
| 2 | 工程适用性 | 制造直觉 (§二) | ray-casting壁厚 + 曲率 + 拔模角 + 边质量 |
| 3 | 装配验证 | 可供性感知 (Gibson) | 干涉/间隙/配合面 |
| 4 | 格式合规性 | 信息传递完整性 | STEP回环 + STL网格质量 |
| 5 | 参数合规性 | 规格对照 | 设计参数/BOM验证 |
| 6 | 设计意图验证 | 2.5D草图 (Marr) | 拓扑亏格 + 面数 + 功能特征匹配 |
| 7 | 人类感知模拟 | 三根柱子的整合 | 主惯性轴 + 对称性 + 稳定性 + 可供性 |

**关键映射**:
- §二 壁厚规则 → L2 `BRepExtrema_DistShapeShape` 实测
- §二 悬垂规则 → L2 法向量 vs 重力分析
- §三 功能→结构 → L6 `key_features` 验证
- §六 Euler公式 → L0 + L6 亏格检查
- §八 五个问题 → L7 全部可回答

CLI入口:
```bash
python forge_v3.py audit <step_file>        # 单文件八层审核
python forge_v3.py audit-dir <directory>     # 批量审核
python dao_audit.py <step_file> --json       # JSON输出
```
