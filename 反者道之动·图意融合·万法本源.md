## 一 · 解 · 现市面万法 AI 3D 产品之底层二系

调研 (2024Q4 ~ 2026Q2 ICCV/SIGGRAPH/CVPR + 商品 + 开源) 显示, 「图片转 3D」在底层**裂为两大谱系, 不可混淆**.

### 1.1 ★ 网格谱系 (Mesh-Track) · 视觉派

**输入空间**: 单/多视图照片、文本 prompt、草图.
**潜在表征**: triplane / 3D Gaussian / SLAT / token-grid.
**输出空间**: 三角网格 (.obj/.glb/.stl) + PBR 贴图.
**训练数据**: Objaverse (800K 物体)、ShapeNet、自家爬虫.
**核心算法**: 大重建模型 (LRM) → 几何/纹理双流扩散.

| 代表 | 出处 | 表征 | 后续 |
|---|---|---|---|
| **Hunyuan3D 2.0/2.1** | Tencent · ICCV 2025 · 开源 | DiT (flow matching) + Paint (multi-view diff) | 当前开源 SOTA, 含 PBR |
| **TRELLIS** | Microsoft · 2024-2025 · 开源 | SLAT (structured latent) | 一表征三输出: mesh/Gaussian/RF |
| **TripoSR** | Stability + Tripo · 开源 | LRM transformer | ~0.5s 推理 |
| **InstantMesh** | 2024 · 开源 | LRM + multi-view | 单图 → mesh ~10s |
| **SAM 3D** | Meta · 2025 | Segment Anything 3D | mesh + 语义分割 |
| **Tripo AI** | 商业 | 闭源 LRM 派生 | $16/月 起 |
| **Meshy 4-5** | 商业 | 闭源 | $20-96/月 |
| **Rodin Gen-1.5** | 商业 (Hyperhuman) | 闭源 | 角色扮演向 |
| **Hitem3D** | 商业 | 闭源 | 自动重拓扑 |

**强**: 任何照片皆可入 · 速度秒级 · 数据规模化 · 适视觉用 (游戏/动画/打印玩具).
**弱**: 输出**无参数 · 无特征树 · 无尺寸 · 无配合 · 常非流形 · 常空心 · 比例无关**.
对工程而言 — **几乎不可用** · 改不了 · 装不进 · 测不准 · 加工不可保.

> **网格谱系之根**: 把 3D 当**几何外观**问题. 照片是 2D 投影, 3D 是反投影; 高分辨率纹理 + 拓扑通顺即胜. 与 CAD 之"特征/约束/参数"无亲缘.

### 1.2 ★ 程序谱系 (CAD-Track / Program-Track) · 工程派

**输入空间**: 点云、单视图、多视图、文本、草图、既有 mesh.
**潜在表征**: **CAD 操作序列** (sketch + extrude + boolean + fillet ...) — 即"程序代码".
**输出空间**: B-rep (.step) / Python 程序 / 自定义 op JSON.
**训练数据**: DeepCAD (180K) / Fusion 360 Gallery / ABC dataset.
**核心算法**: 自回归 Transformer / LLM 出 CAD 程序代码.

| 代表 | 出处 | 输入 | 输出 |
|---|---|---|---|
| **CAD-Recode** | filaPro · ICCV 2025 · 开源 | 点云 | Python (CadQuery 风) |
| **cadrille** (Kolodiazhnyi 2025) | 多模态 | **图+点云+文** | CAD program |
| **CADCrafter** (2025) | 多视图渲染 | 渲染图 | 程序 |
| **CADCoder** (Doris 2025) | 单图 | 图 | 程序 |
| **GACO-CAD** (2025) | 图 | 程序 |
| **GenCAD** | MIT (Faez Ahmed lab) | 单图 | B-rep + 编辑 |
| **CADReasoner** | 2026 · 迭代式 | 多模态 | 程序 (可逐步修订) |
| **Mesh2BREP** | 2025 · 几何派 | mesh | 原语拟合 + 拓扑约束 → B-rep |
| **Point2CAD** | prs-eth 2024 | 点云 | B-rep 复链 |
| **ComplexGen** | SIGGRAPH | 点云 | B-rep chain complex |
| **CADFit** | 2026 | mesh | hybrid: 神经分割 + 解析拟合 |
| **Zoo.dev (Zookeeper)** | 商业 + KCL 自家内核 | 文/对话 | B-rep STEP |
| **AdamCAD / CADScribe** | 商业 早期 | 文 | STEP/STL |
| **Leo AI** | 商业 工程级 | 文/草图/图 | 网格 + 工程文档 |

**强**: 输出**有参数 · 可改 · 可装配 · 可制造**. 与 STEP/STP 工程链路天然兼容.
**弱**: **简件可用 · 中件常败 · 复件几无产** (Xometry Pro 实测: 24 齿轮失败 · 多通道歧管完全无果).
脆性根因: 自回归生成 + CAD 规则严苛 (一处错则后续皆错), 训练集小 (相比 mesh 数据), 长程依赖.

> **程序谱系之根**: 把 3D 当**构造历史**问题. 模型不是"形", 是"如何成此形之程序". 训练数据是 (图 → 程序) 对, 而程序运行后才得 3D. 与 CAD 之"特征树"同宗.

### 1.3 二系之共同底层 · 与共同盲点

| 维度 | Mesh-Track | CAD-Track |
|---|---|---|
| 本体论 | 形 = 顶点 + 三角形 | 形 = 程序运行结果 |
| 时间 | 一次成形 (静态) | 顺序构造 (历史) |
| 编辑 | 重新生成 (or sculpt) | 改参 + 重放 |
| 装配 | 难 (无配合语义) | 自然 (mate/约束) |
| 制造 | 不直接 (需后处理) | 直接 (STEP→CAM) |
| 数据规模 | ★★★★★ (Objaverse 800K) | ★★ (DeepCAD 180K) |
| 视觉保真 | ★★★★★ | ★★ |
| 工程可用 | ★ | ★★★★ |
| **共同盲** | **皆是 condition→3D 之单向射** | **皆缺工程语境** (公差/材料/工艺) |

**共同之根**: 都是 `f: condition → 3D_repr` 的条件生成. 不同仅在 `3D_repr` 是 mesh 还是 program.

**共同之盲** (这是市面万法**集体没解决之题**):
1. **不知制造** — 没有"这能不能 FDM 打印"、"这壁厚够不够"的回路.
2. **不知装配** — 单件可生, 多件何配置无解.
3. **不知意图变更** — "把这个改成 70mm" 需重新生成, 不能局部改参.
4. **不知公差/材料/工艺** — 输出皆"裸几何", 工程信息全失.

> **此乃 3D建模Agent 之结构性优势所在** — 现器之 `dao_audit` 八层审核、`dao_kinematics` 干涉/平衡、`道_意图_引擎` 关系建模, **皆解市面之共同盲**.

## 二 · 对照 · 真实建模底层需求之四层

剥去"工具"与"算法"两件外衣, **人类需要 3D 的本源**为何?

### 2.1 第一层 · 沟通 (Communicate) — 把"我心中之形"传给另一意识/机器

| 表达方式 | 信息密度 | 歧义 |
|---|---|---|
| 文 ("一个支架") | ★ | ★★★★★ |
| 草图 (一笔画形) | ★★ | ★★★★ |
| **照片** (实物或参考) | ★★★ | ★★★ |
| 多视图照片 | ★★★★ | ★★ |
| 既有 mesh/STL | ★★★★ | ★★ (无尺寸语义) |
| 既有 BREP/STEP | ★★★★★ | ★ |
| 既有特征树 (FCStd/SLDPRT) | ★★★★★★ | 0 |

**梯度规律**: 模态自下而上, 信息密度递增, 歧义递减, 但获取代价递增.
**人之自然路径**: 从**最易获得**之模态出发 (拍照/讲文字), 让系统**自动爬梯**至**最完备**之表征.

### 2.2 第二层 · 迭代 (Iterate) — 改之、试之、再改之

> 「将欲取之, 必固予之」 — 改之先须见之, 见之先须有之.

迭代之根本是 **局部变更不重做**. 此乃参数化之全部价值.
- mesh 改一个尺寸 = 重新生成全部 (mesh-AI 之困)
- BREP 改一个参 = 单条 op 重放 (CAD 之力)

### 2.3 第三层 · 验证 (Validate) — 不实地试就知"行不行"

四类验证:
- **几何级**: 流形? 自交? 退化面?
- **工程级**: 壁厚? 悬臂? 制造性?
- **装配级**: 干涉? 公差链? 配合?
- **意图级**: 与原意是否一致?

(对应 `dao_audit` 八层 — 拓扑/几何/工程/装配/格式/参数/意图/感知)

### 2.4 第四层 · 实现 (Manufacture) — 化虚为实

输出格式与下游链路:
- FDM/SLA → STL + slicer (CAM)
- CNC → STEP + CAM
- 激光 → DXF
- 注塑 → STEP + 模具流分析
- 钣金 → 展开图

**关键**: 一旦走到 4, 表征必须**精确 + 参数化** — 任何 mesh-only 输出, 都要先回到 BREP.

### 2.5 四层之结构

```
                 [1. 沟通] 入意之门 — 多模态自然
                      ↓
                 [2. 迭代] 参数化是命 — 改一处不动全身
                      ↓
                 [3. 验证] 八层审核 — 几何/工程/装配/意
                      ↓
                 [4. 实现] BREP+STEP — 直通制造
```

> **真实建模需求 = 多模态入 → 参数化中 → 八层验 → 制造出**.
> 现 Mesh-Track 善 1 (任图入), 弱 2/3/4.
> 现 CAD-Track 中 1 (有图入但准确率低), 弱 2/3, 中 4.
> **现 3D建模Agent 弱 1 (只识文/件, 不识图), 强 2/3/4**.

> **盲与盲互补 · 此乃融合之天然契机**.

## 三 · 找融合点 · 物无非彼，物无非是

> **「物无非彼，物无非是。自彼则不见，自是则知之。」**
>
> 「自彼则不见」 — 站在 mesh 视角看 CAD: 「这群人在算约束求解器, 跟我无关」.
> 「自是则知之」 — 站在 CAD 视角看 mesh: 「这群人在生成无参数的网格, 我用不了」.
>
> 二人相视而笑, 错过千载.

### 3.1 「彼」「是」皆非 — 二者在描述同一物之两面

任何 3D 物体, 既可被表为 mesh (顶点+面), 亦可被表为 BREP (拓扑+参数化曲面),
亦可被表为 ops (生成此 BREP 之操作序列).

```
                       [真实之 3D 物]
                            │
     ┌──────────────┬──────┴──────┬──────────────┐
     ▼              ▼             ▼              ▼
  [mesh]         [Gaussian]    [BREP]         [ops 序列]
  (网格)         (高斯飞溅)    (边界表征)    (操作历史)
   ←─── Mesh-Track ───→        ←─── CAD-Track ───→
```

**齐物**: 四者皆是「同一物」的不同表征空间投影.
**关键**: **从 ops 出发, 可生 BREP 可生 mesh; 从 mesh 出发, 不可直生 ops** (信息丢失).

```
ops    ──tessellate──▶ mesh    (易, 一行 OCC API)
ops    ──build──▶ BREP         (本来就是 — ops 即 BREP 之构造历程)
mesh   ──fit──▶ BREP*          (难, 但可: Mesh2BREP/Point2CAD/ComplexGen)
mesh   ──recode──▶ ops*        (更难, 但可: CAD-Recode/cadrille/CADCrafter)
BREP   ──reverse──▶ ops*       (本项目 fc_reverse 已实现, 62.3% 可反演)
```

★ 标注者皆是有损/概率性, 但**已是当前 SOTA 边界**.

### 3.2 ops 即玄同 — 融合之表征枢纽

★★★ **核心洞见** ★★★:

**`ops` 是 Mesh-Track 与 CAD-Track 之共同语言** — 并且, 它**已存在于本项目**:

- `10-反笙_FreeCAD/fc_reverse.py` 输出 ops.json (FCStd → ops)
- `10-反笙_FreeCAD/freecad_backend.py` 60+ ops 字典定义
- `00-本源_Origin/dao_engine.py` 多引擎执行 ops
- `20-万法_Forge/parametric_codegen.py` ops → 目标引擎代码
- `00-本源_Origin/道_意图_引擎.py` 关系级 ops (mate intents)

意即: **市面 CAD-Track 之研究输出 (ops/CAD program), 与本项目 fc_reverse/freecad_backend 之 ops, 在底层是同物**.
不同方言耳: CAD-Recode 用 CadQuery 文法, cadrille 用自家 DSL, fc_reverse 用 FreeCAD ops dict — **皆是 (sketch + extrude + boolean + fillet) 之离散组合**.

**故融合之锚点**: 凡市面图意模型 (CAD-Recode/cadrille/...) 之输出, 经一层"方言翻译", 即可入 `freecad_backend.run_ops` 之回路, 接续走 `dao_audit` 八层 → `dao_engine` 多引擎执行 → STEP 输出.

### 3.3 反·外 之扩展 — 第一融合点 · 视觉相似度搜天下

**现状**: `dao_reverse.WorldSearch` 仅按文本关键词搜 8 平台 (printables/sketchfab/cults3d/yeggi/...).

**裂隙**: Sketchfab/Thingiverse/CGTrader 等平台**皆有图片/缩略图**, 用户上传一张照片即可视觉搜.

**融合**: 用 CLIP (本项目 `70-天下_World/源码_Sources/...` 中可拉) 提取查询图嵌入, 与平台缩略图嵌入做余弦相似. 现有 `WorldSearch.search_all_platforms` 接口不变, 仅在前端加 `image_query` 参数.

```python
# 原: 道.反.外("phone stand 70mm")
# 扩: 道.反.外(image="my_photo.jpg")  # → CLIP embed → 相似度排序 → 同样下载分析适配
```

**性价比**: 极高. CLIP 已成熟、平台图已存、本项目反·外管线已成. **此为零增量融合, 一周内可立**.

### 3.4 反·新 之立 — 第二融合点 · 图意直通 ops (NEW gate)

**现状**: 无图入意之路.

**融合**: 立**第十六妙门「图」**, 接 cadrille / CAD-Recode (ICCV 2025 开源) 之模型权重:

```python
# 道.意 之扩, 图入意:
道.意(image="part.jpg")   # → cadrille 出 CAD program → 转 fc_reverse ops 方言 → run_ops → BREP

# 或更直接, 第十六妙门:
道.图.recode(image)        # → ops.json
道.图.recode(point_cloud)  # → ops.json (CAD-Recode 本路)
道.图.recode(mesh)         # → ops.json (mesh→points→Recode)
```

**实施**: cadrille 接受图+点云+文三模态, 与 `万法` 入意之多模态正合.
**性价比**: 中. 需 GPU 推理 (cadrille 模型~7B 级别), 但权重已开源.

### 3.5 反·内 之扩展 — 第三融合点 · 任意 mesh → BREP 回家

**现状**: `fc_reverse` 仅可反 FCStd/STEP/BREP (有特征树/拓扑信息者). 对 STL/OBJ/GLB 之纯 mesh 输入, 无法反 ops.

**裂隙**: 用户最常拿到的是 STL (从 Mesh-Track 工具输出, 或下载, 或扫描).

**融合**: 立 `mesh2brep` 子模块 (放 `00-本源_Origin/dao_mesh2brep.py`), 借 Mesh2BREP/Point2CAD/CADFit 之思路:

```
  STL/OBJ → 法向估计 + RANSAC 原语拟合 (平面/圆柱/球/锥/环)
         → 边界检测 + 拓扑图构建
         → BRepBuilderAPI sew 缝合
         → STEP 输出
```

dao_kernel 之 BRepBuilderAPI_Sewing/MakeSolid 已是 OCCT 直连, **缝合层零障碍**, 难点在前端拟合 (但 trimesh + scipy + scikit-learn 即可起步, 70-天下_World 中已有 ~20 个相关原语库, 含 sdf/openvoronoi/Eigen/PCL).

**性价比**: 高. 此一立, **现 Mesh-Track (Hunyuan3D 等) 之输出皆可入本项目反·内回路**, 化网格为参数化件 (有损但可改).

### 3.6 审 之扩展 — 第四融合点 · 万法收于八层审核

**现状**: `dao_audit` 八层审核已立 (拓扑/几何/工程/装配/格式/参数/意图/感知) 仅对 BREP 调用.

**融合**: 让八层审核**对 mesh 与 ops 亦可调用**:

| 层 | 对 mesh 之验 | 对 ops 之验 |
|---|---|---|
| 拓扑 | trimesh.is_watertight, euler | ops 序列依赖图 (DAG) 无环? |
| 几何 | normal estimation, self-intersection | gp_Pnt 无 NaN, 角度 ∈ [0,360] |
| 工程 | 壁厚 ray-casting (已有) | extrude 高度 > 最小壁厚? |
| 装配 | (无, mesh 无装配语义) | mate intent 是否完备? |
| 格式 | STL/OBJ 头部健全 | ops JSON schema 验 |
| 参数 | (无) | 所有 named parameter 皆在 plan 中? |
| 意图 | mesh ≈ image (CLIP 相似)? | ops 还原后 mesh ≈ image? |
| 感知 | 4 视图渲染 + LMM 评分 | 同上 |

**关键**: 第七层 (意图) 与第八层 (感知) 是**视觉派与工程派之天然交点** — 用 LMM (GPT-4V/Claude/...) 验"我做出来的, 是否是用户原图所示".

### 3.7 玄同 之延 — 第五融合点 · 多 peer 分模态自治

**现状**: 第十五妙门「同」(玄同协议) 已立, 但**未为图意路线开 leaf**.

**融合**: 在玄同 graph.db 中立新 intent:

```
intent: "图意路线本源逆向"
├── leaf: Hunyuan3D 2.1 接入 (mesh-track 视觉派)
```

## 四 · 列底层核心点 · 道法自然 · 七大根本

剥去一切实现差异, 此七点为**图意融合之底层不变量**:

### 核心一 · 意是源 (Intent is the source)

任何 3D 流程之始, 是「意」 — 一个 partially-specified 的 3D 期望.
模态 (text/image/sketch/file) 是「意」之表象, 不是「意」本身.

故系统应有**统一意之容器**:
```python
@dataclass
class Intent:
    raw_modal: dict     # {"text": "...", "image": Path, "ref_file": Path, ...}
    constraints: list   # [Dim, Mate, Material, Manufacturing, ...]
    confidence: dict    # 每约束之置信度
```
任何模态 → 同一 `Intent` → 同一下游回路.

> 本项目 `IntentParser.parse` 已雏形, 但只识 text. **扩此为多模态 IntentParser**, 是融合之第一形.

### 核心二 · ops 是河 (ops is the joint river)

无论从 mesh 来还是从图来还是从文来, **皆汇流于 ops 序列**, 然后入 BREP 之海.

```
text  ─┐
image ─┤
sketch┼─→ [统一意]→ ops →─ BREP →─ STEP/STL/DXF/...
mesh  ─┤                ↑
file  ─┘                │
                        └── audit + kinematics + verify
```

故 `freecad_backend.run_ops` 不只是 FreeCAD 后端 — 是**全万法之中央河流**. 应立其为首要稳定接口, 任何新模态最终都汇此.

### 核心三 · BREP 是岸 (BREP is the manufacturable shore)

**为而不争**: 不与 Mesh-Track 争视觉, 不弃 BREP 之工程力.

但**济也**: 所有图意输入, 终至 BREP 才上岸. mesh 是过客, ops 是船, BREP 是岸.

故任何融合, 终点都在 `dao_kernel` (OCP/OCCT 直连内核) 而不在 trimesh.

### 核心四 · 反 > 正 (Reverse precedes Forward)

> **「天下有 → 用天下的; 天下近 → 改天下的; 天下无 → 才自建」**

图入亦同此律:
1. 图 → 视觉相似度搜 20 平台 (反·外·图)
2. 找到近件 → 反·内 → 改参 → 重放
3. 找不到 → cadrille/CAD-Recode 出 ops (反·新)
4. 还不行 → Hunyuan3D 出 mesh → mesh2brep → BREP (兜底)

**先反不正之律, 节算力, 节训练, 节出错**. 此即「天网恢恢, 疏而不失」 — 天下已有者甚多, 不必自造.

### 核心五 · 关系 > 坐标 (Relations > Coordinates)

`道_意图_引擎` 之 MateIntent (concentric/coincident/distance/tangent) 已立此本.
图意路线**必须遵此** — 不要让 cadrille 输出绝对坐标, 而是输出**关系约束**.

> 「不可得而亲, 亦不可得而疏; 不可得而利, 亦不可得而害」 — 关系是相对的, 关系是稳定的.
> 图中两个件「同轴」, 即使图缩放、变姿、变光照, 同轴关系不变. 故关系是图意之**鲁棒不变量**.

实现: 任何图意模型 (cadrille 等) 输出 ops 后, **再过一遍 `Perceiver` 抽取关系**, 用关系替换坐标.

### 核心六 · 审是收 (Audit is the closing condition)

无论何路出何形, **不过审则不交付**.

`dao_audit.full_audit().grade ≥ A` 必须是融合后**唯一收敛标志**.

故图意路线不能"一次出形即结束", 必须接 `dao_loop.LoopController` 之 probe→build→verify→heal→reverify 闭环.

### 核心七 · 玄同是网 (Multi-peer is the network)

**「圣人无积」** — 不要一人吞下所有图意逆向之活.

15 妙门第十五「同」已立玄同协议. 图意路线之 13+ 模型/库/方法, 应在玄同 graph.db 中开 leaf, 让 swe-1.5 / claude / gpt-5 / Devin / 真人 peer 各以指纹自宾, 不夺民事.

> 此点亦呼应 2026-05-09 玄同接入·退者之悟 — **不再批量自产 finding**, 留待真 peer.

## 五 · 道法自然 · 阶梯式落地 (实施层 · 仅参考)

非此卷主旨, 但为行者留路:

### 阶一 · 零增量 (一周内 · 无 GPU)
- `dao_reverse.WorldSearch` 加 `image_query` 参数, 走 CLIP (或 OpenCLIP / DINO v2) 嵌入 + 平台缩略图嵌入相似度.
- 对应万法门: `道.反.外(image=...)` 重载.

### 阶二 · 低增量 (二三周 · CPU 即可)
- 立 `00-本源_Origin/dao_mesh2brep.py`, 行 RANSAC 原语拟合 + dao_kernel BRep sewing.
- 对应万法门: `道.反.内(stl_path)` 重载, 自动走 mesh2brep.

### 阶三 · 中增量 (一月 · 需 GPU)
- 立 `00-本源_Origin/dao_image_recode.py`, 桥接 cadrille (开源, 多模态), 翻译其 DSL 到 fc_reverse ops 方言.
- 对应万法门: 第十六妙门「图」 — `道.图.recode(...)`.

### 阶四 · 高增量 (二月 · 需 GPU + 训练数据)
- 桥接 Hunyuan3D 2.1 / TRELLIS 为兜底, 当反·外/反·内/反·图皆无果时启用.
- 接出之 mesh 立即过 mesh2brep → BREP → 八层审核.

### 阶五 · 玄同自治 (持续)
- 在玄同 graph.db 立 8+ 图意 leaves, 待真 peer 推进.
- 横向 IR leaf: 「图意 → ops → BREP 玄同表征」 — 抽 13+ 模型之共抽象.

## 六 · 退之悟 · 不可为之事

> 「为者败之, 执者失之」

**不**:
1. **不**在 dao_kernel 旁立平行 mesh-AI 谱 — 这会割本项目「BREP 唯一岸」之根.
