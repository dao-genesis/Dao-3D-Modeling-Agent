# ToyCar v3 · 通用经验沉淀（可迁移到任何装配工程）

> 从 v3 全流程（建模 → 0 干涉 → 动画验证 → 水密自审 → 底层算法修复）提炼的通用准则。

## 1. 几何算法：圆弧扫角必须"短程 CCW"

渐开线齿轮轮廓中齿顶弧/齿根弧的扫角若用 `if end < start: end += 2π` 补角，
浮点误差会让弧绕远路（甚至多圈），轮廓自交 → 体积虚高（SpurGear 曾虚高 27 倍）→ BREP 无效。

**准则**：任何按极角采样的圆弧一律 `Δ = (end - start) % 2π` 取短程 CCW 扫角；
生成实体后立即 `shape.isValid()`，无效则 `shape.fix()` 再验，仍无效即抛错——绝不让坏几何下行。

## 2. 自审三道门（每次装配变更后必跑）

| 门 | 手段 | 通过标准 |
|---|---|---|
| 干涉 | 逐对 `Part.Shape.common()` | 全对体积交集为空 |
| 水密 | STL 导出 + `trimesh.is_watertight` | 15/15（全件）True |
| 运动 | GUI 动画（爆炸 / 传动仿真）录屏 | 无穿模、啮合不脱离 |

水密门是干涉门抓不到的：自交轮廓拉伸出的实体可以"看似正常"且 common() 正常，
但网格化后非流形——**它是唯一暴露了齿轮底层缺陷的门**。

## 3. 质量核算随几何联动

体积虚高直接导致质量虚高（592g → 修复后 228.8g）。工程 JSON 里的 mass/性能参数
必须由最终 BREP 重算生成，不许手填。

## 4. 修复顺序：底层算法 → 重生零件 → 全门复审

发现坏零件时不修零件本身（remesh/补洞是治标），而是回溯生成算法修根，
再重生零件、重跑全部三道门确认回归为零。
